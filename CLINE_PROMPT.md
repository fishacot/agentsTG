## Контекст проекта

Ты работаешь над **мультиагентной системой для Telegram** — персональным AI помощником, который автоматизирует личную жизнь пользователя через специализированных агентов.

### Цель проекта
Создать Telegram бота с 5 AI агентами:
1. **Coordinator** - маршрутизация запросов
2. **Planner** - календарь, задачи, напоминания
3. **Finance** - учёт расходов, бюджет, аналитика
4. **Notes** - заметки, knowledge base
5. **Integration** - внешние API (бронирование, календари, банки)

### Технологический стек
- **Backend:** Python 3.11+ с aiogram 3.x
- **БД:** PostgreSQL + Redis
- **AI:** Qwen2.5-72B-Instruct через Alibaba Cloud DashScope API
- **Хостинг:** Railway.app (бесплатный tier, 24/7)
- **Инфраструктура:** Celery, Alembic, FastAPI

### Ограничения
- Пользователь на ноутбуке - локальные модели не подходят
- Нужен бесплатный хостинг 24/7
- Важна автономность и сохранение контекста
- Работа в Harvi Code с ограниченными лимитами

## 📋 Обязательный workflow (CursoRules)

### Перед началом ЛЮБОЙ задачи:

1. **Прочитай файлы состояния:**
   ```
   - PROJECT_STATE.json (текущая фаза, задачи, архитектура)
   - PROGRESS.md (что сделано, что в процессе)
   - docs/implementation-notes.md (журнал решений)
   - PROJECT_PROMPT.md (архитектура, промпты агентов)
   ```

2. **Пойми контекст:**
   - Какая сейчас фаза проекта?
   - Что уже реализовано?
   - Какие есть зависимости?
   - Есть ли блокеры?

### Рабочий цикл:

```
1. ПЛАН
   ↓ Что меняется, какие файлы, без лишнего
   
2. РЕАЛИЗАЦИЯ
   ↓ Минимальный diff, не трогать несвязанное
   
3. ПРОВЕРКА
   ↓ docs/PROJECT_VERIFICATION.md - команды проверки
   ↓ До 3 попыток исправить, потом стоп и вопрос
   
4. ЖУРНАЛ
   ↓ Обновить docs/implementation-notes.md
   ↓ Решения вне spec, tradeoffs, результаты verify
   
5. ИТОГ
   ↓ Что сделано, где смотреть
   ↓ Обновить PROJECT_STATE.json и PROGRESS.md
```

### После каждой задачи:

1. **Обновить журнал** `docs/implementation-notes.md`:
   ```markdown
   ## YYYY-MM-DD — Краткий заголовок
   - **Контекст:** задача
   - **Решение:** что сделано
   - **Не в SPEC:** отклонения и почему
   - **Tradeoffs:** компромиссы
   - **Файлы:** список изменённых файлов
   - **Проверки:** команды и результат
   - **Следующий шаг:** что делать дальше
   ```

2. **Обновить состояние** `PROJECT_STATE.json`:
   - Переместить задачу из `current_tasks` в `completed_tasks`
   - Добавить новые задачи в `next_tasks` если нужно

3. **Обновить прогресс** `PROGRESS.md`:
   - Отметить выполненные пункты `[x]`
   - Обновить метрики

- ✅ Простота > сложность
- ✅ Читаемость > краткость
- ✅ Type hints везде
- ✅ Docstrings для публичных методов
- ✅ Async/await для I/O

- ❌ Не делать функции > 50 строк

### Тестирование
**Обязательно перед коммитом:**
```bash
black . && isort . && flake8 src/ && pytest tests/ -v
```

### Документация
- Обновлять после каждой значимой задачи
- Фиксировать все решения и tradeoffs
- Не дублировать длинные правила в ответах

Формат: `<type>(<scope>): <subject>`

Типы: `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`

Пример:
```
feat(planner): добавлен метод create_event

- Реализован базовый функционал создания событий
- Добавлены тесты
- Обновлена документация

Closes #5
```
