# Канал заметок Эльзы (Telegram)

Приватный канал, куда Эльза публикует заметки через инструмент `post_to_notes_channel`. Бюджет API = 0 (только Bot API).

## Шаги

1. **Создайте канал** в Telegram (Private channel).
2. **Добавьте бота Эльзы** (`@elliza_pa_bot` или ваш `BOT_TOKEN_PA`) как **администратора** с правом **Post messages**.
3. **Узнайте chat_id канала:**
   - перешлите любое сообщение из канала боту [@getidsbot](https://t.me/getidsbot), или
   - опубликуйте тестовое сообщение и вызовите `getUpdates` Bot API.
4. **Добавьте в `.env` на VPS:**

   ```env
   NOTES_CHANNEL_ID=-1001234567890
   ```

   ID каналов обычно начинается с `-100`.

5. **Перезапустите сервис:**

   ```bash
   cd /opt/agentsTG && git pull && systemctl restart agents-tg
   ```

## Поведение

- Если `NOTES_CHANNEL_ID` **не задан** — инструмент вернёт `notes_channel_not_configured`; Эльза использует только Obsidian vault.
- Если задан — при явной просьбе «запиши заметку» модель может выбрать vault, канал или оба.
- Сообщения отправляются в **HTML** (`<b>заголовок</b>` + текст).

## Проверка

1. В ЛС Эльзе: «Запиши заметку: тест канала — проверка публикации».
2. Убедитесь, что сообщение появилось в канале.
3. В vault (если настроен `OBSIDIAN_VAULT_PATH`) — опционально второй файл.

## Troubleshooting

| Симптом | Решение |
|---------|---------|
| `Chat not found` | Бот не добавлен в канал или неверный `NOTES_CHANNEL_ID` |
| `Forbidden` | У бота нет права публиковать сообщения |
| Заметка только в vault | Канал не настроен или модель выбрала только vault — нормально |

См. также: `OBSIDIAN_VAULT_PATH`, `deploy/TIMWEB_VPS_GUIDE.md`.
