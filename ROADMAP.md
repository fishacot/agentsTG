## Этап 1: P0 — Критические фиксы (Неделя 1)
**Цель:** Сделать бота стабильным для ежедневного использования

### 1.1 Callback handlers для inline-кнопок
- **Задача:** Реализовать обработку нажатий кнопок меню
- **Файлы:** `src/agents_tg/bot/handlers/callbacks.py`, `src/agents_tg/bot/__init__.py`
- **Критерий:** Нажатие на кнопку агента вызывает соответствующего агента
- **Время:** 2-3 часа

### 1.2 Подключение PostgreSQL
- **Задача:** Инициализация БД в `on_startup`, graceful shutdown в `on_shutdown`
- **Файлы:** `src/agents_tg/db/session.py`, `src/main.py`
- **Критерий:** Бот стартует с подключением к БД, создает таблицы при первом запуске
- **Время:** 3-4 часа

### 1.3 Rate limiting (защита от флуда)
- **Задача:** Middleware для ограничения запросов (3 msg/min на пользователя)
- **Файлы:** `src/agents_tg/bot/middlewares/ratelimit.py`
- **Критерий:** Флудер получает "Слишком быстро, подождите 30 сек"
- **Время:** 2-3 часа

### 1.4 Удаление мертвого кода
- **Задача:** Удалить старый `coordinator.py`, неиспользуемые импорты
- **Файлы:** `src/agents_tg/agents/coordinator.py` (удалить), cleanup других файлов
- **Критерий:** `flake8` без ошибок, нет # noqa без причины
- **Время:** 1-2 часа

**Итого Этап 1:** 8-12 часов | **Готовность к использованию:** Да

## Этап 2: P0 — Инфраструктура стабильности (Неделя 2)
**Цель:** Бот не падает, переживает рестарты, мониторится

### 2.1 Webhook mode (альтернатива polling)
- **Задача:** FastAPI endpoint `/webhook` + переключатель POLLING_MODE
- **Файлы:** `src/agents_tg/bot/webhook.py`, `.env.example`
- **Критерий:** `WEBHOOK_URL` в env → бот работает на webhook
- **Время:** 4-5 часов

### 2.2 Health checks
- **Задача:** Endpoint `/health` с проверкой БД и Redis
- **Файлы:** `src/agents_tg/api/health.py`
- **Критерий:** Render.com видит статус, рестартует при падении
- **Время:** 2-3 часа

### 2.3 Graceful shutdown
- **Задача:** Закрытие соединений БД, Redis, HTTP сессий при SIGTERM
- **Файлы:** `src/main.py` — `on_shutdown`
- **Критерий:** 0 потерянных соединений при рестарте
- **Время:** 2-3 часа

### 2.4 Logging & Error tracking
- **Задача:** Структурированные логи JSON, уровни DEBUG/INFO/ERROR
- **Файлы:** `src/agents_tg/utils/logger.py`
- **Критерий:** Логи читаются в Render Dashboard, видны ошибки агентов
- **Время:** 3-4 часа

**Итого Этап 2:** 11-15 часов | **Готовность к продакшену:** Базовая

---

## Этап 3: P1 — Архитектура данных (Недели 3-4)
**Цель:** RAG, кэширование, быстрые ответы

### 3.1 Векторная БД (Chroma/Qdrant)
- **Задача:** Интеграция Chroma для RAG, хранение эмбеддингов диалогов
- **Файлы:** `src/agents_tg/services/vector_store.py`
- **Критерий:** Агенты ищут похожие запросы из истории, ответы быстрее
- **Время:** 6-8 часов

### 3.2 Redis кэширование
- **Задача:** Кэш DuckDuckGo результатов (TTL 1 час), кэш ответов LLM (TTL 24 часа)
- **Файлы:** `src/agents_tg/services/cache.py`
- **Критерий:** Повторный запрос = ответ < 1 сек, нет лишних API вызовов
- **Время:** 4-5 часов

### 3.3 Фоновые задачи (Celery/RQ)
- **Задача:** Отправка длинных задач в очередь, бот отвечает сразу "Думаю..."
- **Файлы:** `src/agents_tg/worker/tasks.py`, `src/agents_tg/bot/handlers.py`
- **Критерий:** Тяжелый research запускается в фоне, пользователь получает уведомление
- **Время:** 8-10 часов

**Итого Этап 3:** 18-23 часа | **Готовность:** Улучшенная UX

---

## Этап 4: P1 — Потоковые ответы (Неделя 5)
**Цель:** Пользователь видит "печатание" агента, не ждет 30 сек молча

### 4.1 Streaming для LLM
- **Задача:** Поддержка streaming от HuggingFace/OpenRouter
- **Файлы:** `src/agents_tg/services/qwen_client.py`
- **Критерий:** Токены приходят по частям, бот обновляет сообщение каждые 2 сек
- **Время:** 4-6 часов

### 4.2 Прогресс для multi-step задач
- **Задача:** Оркестратор показывает "Шаг 1/3: Research..." → "Шаг 2/3: Analysis..."
- **Файлы:** `src/agents_tg/agents/orchestrator.py`, бот обновляет сообщение
- **Критерий:** Пользователь видит, на каком этапе план
- **Время:** 6-8 часов

**Итого Этап 4:** 10-14 часов | **Готовность:** Профессиональный UX

## Этап 5: P1 — Реальные интеграции (Недели 6-7)
**Цель:** Агенты действуют, а не только советуют

### 5.1 Google Calendar API
- **Задача:** OAuth2, создание/чтение событий, синхронизация
- **Файлы:** `src/agents_tg/integrations/google_calendar.py`
- **Критерий:** "Создай встречу на 15:00" → реальное событие в Google Calendar
- **Время:** 8-12 часов

### 5.2 GitHub API
- **Задача:** Чтение репо, создание issues, PR комментарии (OAuth)
- **Файлы:** `src/agents_tg/integrations/github.py`
- **Критерий:** Coder агент анализирует реальный код из репо
- **Время:** 6-8 часов

### 5.3 Email/Notifications
- **Задача:** Отправка summary на email, уведомления о задачах
- **Файлы:** `src/agents_tg/integrations/notifications.py`
- **Критерий:** Ежедневный digest в 9:00 с задачами и календарем
- **Время:** 4-6 часов

**Итого Этап 5:** 18-26 часов | **Готовность:** Полезный ассистент

## Этап 6: P2 — Безопасность и sandbox (Неделя 8)
**Цель:** Coder агент безопасен, нет утечек данных

### 6.1 E2B Sandbox для кода
- **Задача:** Выполнение кода в изолированном контейнере
- **Файлы:** `src/agents_tg/services/sandbox.py`
- **Критерий:** Coder агент запускает Python код безопасно, видит результат
- **Время:** 6-8 часов

### 6.2 Input validation (Pydantic)
- **Задача:** Все входящие данные валидируются через схемы
- **Файлы:** `src/agents_tg/schemas/`, middleware validation
- **Критерий:** Невалидные данные отклоняются до обработки агентом
- **Время:** 4-6 часов

### 6.3 Secrets management
- **Задача:** Шифрование токенов в БД, ротация ключей
- **Файлы:** `src/agents_tg/utils/crypto.py`
- **Критерий:** API ключи не в plain text, есть команда `/rotate_keys`
- **Время:** 4-5 часов

**Итого Этап 6:** 14-19 часов | **Готовность:** Enterprise-grade security

## Этап 7: P2 — Тестирование и CI/CD (Неделя 9)
**Цель:** Код не ломается, баги ловятся до продакшена

### 7.1 Unit тесты для агентов
- **Задача:** Тесты для каждого агента (mock LLM, mock tools)
- **Файлы:** `tests/unit/test_*.py`
- **Критерий:** Покрытие > 70%, тесты проходят за < 30 сек
- **Время:** 12-16 часов

### 7.2 Integration тесты
- **Задача:** Тестирование с реальными API (rate limited)
- **Файлы:** `tests/integration/`
- **Критерий:** Проверка полного flow: бот → агент → поиск → ответ
- **Время:** 8-10 часов

### 7.3 GitHub Actions CI/CD
- **Задача:** Авто-тесты на PR, авто-деплой на Render
- **Файлы:** `.github/workflows/ci.yml`, `.github/workflows/deploy.yml`
- **Критерий:** Push в main → тесты → деплой без ручных действий
- **Время:** 6-8 часов

**Итого Этап 7:** 26-34 часа | **Готовность:** Профессиональная разработка

## Этап 8: P2 — Мониторинг и аналитика (Неделя 10)
**Цель:** Видно, как используют бота, где ошибки

### 8.1 Метрики и dashboard
- **Задача:** Prometheus метрики: latency, errors, token usage
- **Файлы:** `src/agents_tg/utils/metrics.py`
- **Критерий:** Дашборд с графиками: запросы/час, среднее время ответа
- **Время:** 6-8 часов

### 8.2 Alerting
- **Задача:** Уведомления в Telegram при ошибках/даунтайме
- **Файлы:** `src/agents_tg/utils/alerts.py`
- **Критерий:** Бот падает → сообщение админу через 1 минуту
- **Время:** 4-5 часов

### 8.3 User analytics
- **Задача:** Сколько активных пользователей, популярные агенты
- **Файлы:** `src/agents_tg/analytics/`
- **Критерий:** Отчет: "Coder использовался 45 раз, среднее время ответа 8 сек"
- **Время:** 4-6 часов

**Итого Этап 8:** 14-19 часов | **Готовность:** Observable system

## Этап 9: Фичи и масштабирование (Недели 11-12)
**Цель:** Веб-интерфейс, многопользовательность, enterprise

### 9.1 Web dashboard (минимальный)
- **Задача:** Список диалогов, история, управление агентами
- **Файлы:** `src/agents_tg/web/`, FastAPI статика
- **Критерий:** Открываешь URL → видишь свои задачи и заметки
- **Время:** 12-16 часов

### 9.2 Multi-user с изоляцией
- **Задача:** Каждый пользователь — свои агенты, настройки, память
- **Файлы:** `src/agents_tg/auth/`, middleware user isolation
- **Критерий:** User A не видит данные User B
- **Время:** 8-10 часов

### 9.3 Custom agents (пользовательские)
- **Задача:** Пользователь создает своего агента с SOUL через UI
- **Файлы:** `src/agents_tg/agents/custom.py`
- **Критерий:** "Создай агента для моего бизнеса" → новый агент в меню
- **Время:** 10-12 часов

**Итого Этап 9:** 30-38 часов | **Готовность:** Product-market fit

## Как работать с планом

1. **Берем этап целиком** — не распыляемся
2. **Каждая задача = отдельная сессия** — с чеклистом и verify
3. **После каждого этапа — тестирование** — не идем дальше пока не стабильно
4. **Ведем `implementation-notes.md`** — записываем решения и tradeoffs
